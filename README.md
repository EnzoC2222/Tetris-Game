# Tetris Game 規格說明

## 1. 專案名稱

Tetris Game

## 2. 專案目的

本專案使用 Python 與 Pygame 製作一款可遊玩的俄羅斯方塊遊戲。玩家可以控制方塊移動、旋轉、快速下降，並透過消除完整橫列獲得分數。遊戲同時加入 Hold 保存方塊、Next 3 預覽方塊、Level 加速等功能，讓玩法更接近正式版俄羅斯方塊。

## 3. 開發技術

* 程式語言：Python
* 遊戲套件：Pygame
* 版本控制：Git / GitHub
* 執行方式：本機 Python 環境執行

## 4. 遊戲畫面規格

遊戲畫面分為兩個區域：

### 4.1 主遊戲區

主遊戲區為 10 欄 × 20 列的方格區域，每一格大小為 30 × 30 pixels。

規格如下：

* 欄數：10 columns
* 列數：20 rows
* 單格大小：30 pixels
* 主畫面寬度：300 pixels
* 主畫面高度：600 pixels

玩家控制的方塊會在此區域中下降、移動、旋轉與固定。

### 4.2 側邊資訊區

側邊區域用來顯示遊戲狀態與輔助資訊，包括：

* Score：目前分數
* Level：目前等級
* Lines：已消除行數
* Hold：目前保存的方塊
* Next 3：接下來三個即將出現的方塊
* 操作提示

## 5. 方塊種類

遊戲包含標準俄羅斯方塊的七種方塊：

| 方塊 | 說明     |
| -- | ------ |
| I  | 長條形方塊  |
| O  | 正方形方塊  |
| T  | T 字形方塊 |
| S  | S 字形方塊 |
| Z  | Z 字形方塊 |
| J  | J 字形方塊 |
| L  | L 字形方塊 |

每一種方塊皆使用二維陣列表示，例如：

```python
[[1, 1, 1, 1]]
```

代表 I 方塊。

## 6. 玩家操作

| 按鍵    | 功能             |
| ----- | -------------- |
| ←     | 向左移動           |
| →     | 向右移動           |
| ↓     | 加速下降           |
| ↑     | 旋轉方塊           |
| Space | 直接落到底部         |
| C     | Hold 保存 / 切換方塊 |
| R     | 重新開始遊戲         |

原本曾使用 Shift 作為 Hold 鍵，但因為可能與輸入法切換衝突，因此改為 C 鍵。

## 7. 遊戲核心規則

### 7.1 方塊生成

遊戲開始時會產生目前方塊，並建立三個 Next Queue 方塊。當目前方塊落地並固定後，系統會從 Next Queue 取出下一個方塊，並補上一個新的隨機方塊。

### 7.2 方塊移動

玩家可以控制目前方塊左右移動。每次移動前，系統會檢查該位置是否合法。

不合法情況包括：

* 超出左邊界
* 超出右邊界
* 超出底部
* 與已固定的方塊重疊

### 7.3 方塊旋轉

玩家按下上方向鍵時，目前方塊會順時針旋轉。若旋轉後的位置不合法，方塊會回復到旋轉前的狀態。

### 7.4 方塊固定

當方塊無法再往下移動時，該方塊會固定在遊戲板上，成為 board 的一部分。

### 7.5 消除行

當某一橫列的所有格子都被方塊填滿時，該列會被消除。消除後，上方的方塊會往下補位。

### 7.6 遊戲結束

當新的方塊生成後，如果出生位置已經被其他方塊佔據，遊戲進入 Game Over 狀態。

玩家可以按 R 重新開始。

## 8. Hold 系統規格

玩家可以按 C 保存目前方塊。

Hold 規則如下：

* 每個方塊落地前只能 Hold 一次
* 如果 Hold 區是空的，當前方塊會進入 Hold 區，並從 Next Queue 取出下一個方塊
* 如果 Hold 區已有方塊，當前方塊會與 Hold 方塊交換
* 被 Hold 或交換出來的方塊會重置到初始生成位置
* 當目前方塊固定後，Hold 權限會重置

此設計避免玩家無限切換方塊。

## 9. Next 3 系統規格

遊戲右側會顯示接下來三個方塊。

Next Queue 規則如下：

* 遊戲中永遠維持三個預覽方塊
* 當目前方塊固定後，第一個 Next 方塊會成為新的目前方塊
* 系統會再隨機生成一個新方塊補到 Queue 最後面

此功能讓玩家可以提前規劃落點與策略。

## 10. 分數系統

目前分數計算方式如下：

| 行為            | 分數   |
| ------------- | ---- |
| 按 ↓ 讓方塊下降一格   | +1   |
| 使用 Space 直接落下 | +10  |
| 消除 1 行        | 100  |
| 消除 2 行        | 400  |
| 消除 3 行        | 900  |
| 消除 4 行        | 1600 |

消行分數公式：

```python
score += cleared * cleared * 100
```

因此一次消除越多行，分數成長越高。

## 11. Level 系統

Level 會根據玩家累積消除行數提升。

規則如下：

```python
level = lines // 10 + 1
```

也就是：

| 消除行數  | Level   |
| ----- | ------- |
| 0–9   | Level 1 |
| 10–19 | Level 2 |
| 20–29 | Level 3 |
| 30–39 | Level 4 |

Level 提升後，方塊下降速度會變快。

下降速度計算方式：

```python
fall_speed = max(80, 600 - (level - 1) * 75)
```

代表：

| Level | 下降間隔     |
| ----- | -------- |
| 1     | 600 ms   |
| 2     | 525 ms   |
| 3     | 450 ms   |
| 4     | 375 ms   |
| 5     | 300 ms   |
| 6     | 225 ms   |
| 7     | 150 ms   |
| 8 以上  | 最快 80 ms |

## 12. 程式結構

主要程式集中於 `main.py`。

重要結構如下：

### 12.1 Piece Class

負責表示一個方塊物件。

包含：

* 方塊種類 index
* 方塊形狀 shape
* 方塊顏色 color
* 方塊位置 x, y
* 旋轉功能 rotate()
* 重置位置 reset_position()

### 12.2 Board

Board 是一個 20 × 10 的二維陣列。

空格使用 `None` 表示，已固定方塊使用顏色資料表示。

### 12.3 主要函式

| 函式                | 功能             |
| ----------------- | -------------- |
| create_board()    | 建立空白遊戲板        |
| create_queue()    | 建立 Next 3 方塊佇列 |
| get_next_piece()  | 取得下一個方塊        |
| valid_position()  | 檢查方塊位置是否合法     |
| lock_piece()      | 固定方塊到 board    |
| clear_lines()     | 消除已填滿的行        |
| draw_board()      | 繪製遊戲板          |
| draw_piece()      | 繪製目前方塊         |
| draw_ui()         | 繪製側邊資訊         |
| hard_drop()       | 讓方塊直接落到底       |
| hold_piece_func() | 處理 Hold 方塊     |
| calculate_speed() | 根據 Level 計算速度  |
| reset_game()      | 重置遊戲狀態         |

## 13. 遊戲流程

遊戲主迴圈流程如下：

1. 啟動 Pygame
2. 建立遊戲畫面
3. 初始化 board、目前方塊、Next Queue、分數與 Level
4. 每一幀讀取玩家輸入
5. 根據玩家輸入移動、旋轉、Hold 或 Hard Drop
6. 根據時間自動讓方塊下降
7. 若方塊不能下降，將其固定
8. 檢查是否有完整橫列可以消除
9. 更新分數、行數、Level 與速度
10. 生成下一個方塊
11. 判斷是否 Game Over
12. 重複遊戲直到玩家關閉視窗
